VsCode提交代码显示在签出前，请清理存储工作树

发表于2023-10-08|更新于2023-10-08
|阅读量:

方法一：有如下三步

1.使用 git stash命令将当前自己所修改的代码暂存，并将最近提交的远程仓库内容拉取到本地，将本地代码与上次工作区的内容一致，同时将当前工作区的内容保存到git栈中。

2.使用 git pull拉取最新的仓库内容。

3.使用git stash pop将获取git栈顶的最近一次保存的内容，并合并到代码中，可能会发生代码冲突，在vscode中会在修改过的代码中出现几个选项：
(1)Accept Current Change：使用远程仓库的代码覆盖本地代码
(2)Accept Incoming Change：使用本地代码覆盖远程仓库的代码
(3)Accept Both Change：对两个冲突的代码不做修改，均放入文件内

(4)Compare Changes：对比两段代码

方法二：直接覆盖

1.git reset --hard：强制重置代码，此时不可回退，需要使用此操作时注意做好备份

2.git pull：拉取代码
